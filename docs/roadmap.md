# 로드맵

## 1단계: 지도 구축 가능성 검증

- 코엑스 내부 전체를 최종 범위로 정의합니다.
- 첫 수집은 지하층의 제한된 파일럿 구간에서 시작합니다.
- Polycam 스캔 결과에서 키프레임 이미지, 카메라 자세, 깊이 또는 3D 데이터를 확보할 수 있는지 확인합니다.
- 스캔 좌표계를 평면도 좌표계로 정렬하는 절차를 문서화합니다.

## 2단계: 이미지 검색 베이스라인

- 키프레임 이미지마다 전역 임베딩을 미리 계산합니다.
- 신규 이미지 1장에 대해 후보 위치 top-K를 검색합니다.
- DINOv2-small을 기본 베이스라인으로 사용합니다.
- 검색 결과는 위치 후보, 유사도, 구역 정보를 함께 저장합니다.
- Cloudflare Worker는 Vectorize 검색 API를 담당하고, DINOv2 query 임베딩 생성은 서버 또는 네이티브 앱 경로에서 처리합니다.

## 3단계: 기하 검증

- top-K 후보에 대해 로컬 특징 매칭을 수행합니다.
- XFeat와 LightGlue 조합을 우선 검토합니다.
- 매칭 수, inlier 비율, 재투영 오차를 기준으로 후보를 재정렬합니다.
- 반복 구조가 많은 복도와 매장가에서 오탐을 줄이는지 평가합니다.

## 4단계: 2D-3D 위치추정

- 스캔 또는 재구성 데이터에서 3D 포인트와 키프레임 관측을 연결합니다.
- 사용자 이미지의 2D 특징과 3D 포인트를 매칭합니다.
- PnP와 RANSAC으로 카메라 위치와 방향을 계산합니다.
- 결과를 코엑스 평면도 좌표로 변환합니다.

## 5단계: 전체 코엑스 확장

- 지하층부터 주요 연결 통로와 몰 내부로 구역을 확장합니다.
- 구역별 지도 조각을 공통 좌표계로 병합합니다.
- 조명 변화, 인파, 계절 장식, 매장 변경에 대응하는 재수집 주기를 정의합니다.
- 실패 감지와 후보 다중 반환 정책을 운영 기준으로 정리합니다.
