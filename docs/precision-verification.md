# 정밀 검증 설계

## 왜 필요한가

DINOv2 같은 전역 임베딩 검색은 비슷한 장소 후보를 빠르게 찾는 데 유리합니다. 하지만 코엑스 내부처럼 반복 복도, 비슷한 매장 전면, 대형 광고판이 많은 공간에서는 전역 검색만으로 1-3m 정확도를 보장하기 어렵습니다.

정밀 위치추정에는 top-K 후보를 다시 검증하는 단계가 필요합니다.

## 권장 파이프라인

```text
1. DINOv2-small 또는 동급 모델로 top-K 후보 검색
2. query 이미지와 후보 keyframe 사이 로컬 특징 추출
3. XFeat, LightGlue, SuperPoint, ALIKED 중 하나로 매칭
4. RANSAC으로 기하 일관성 검증
5. 충분한 2D-3D 대응점이 있으면 PnP로 카메라 자세 추정
6. 스캔 좌표를 코엑스 평면도 좌표로 변환
```

## 2단계 운영안

브라우저 또는 Worker는 전역 검색까지만 담당합니다. 정밀 검증은 별도 네이티브 또는 Python 서비스로 분리합니다.

```text
Cloudflare Worker
- 인증, 요청 제한, top-K 벡터 검색, 결과 캐싱

정밀 검증 서비스
- 로컬 특징 추출
- 후보별 매칭
- PnP/RANSAC
- 신뢰도 계산
```

Cloudflare Workers CPU 환경만으로 로컬 특징 매칭과 PnP까지 모두 처리하는 것은 현재 구조에서는 적합하지 않습니다.

## 판정 기준

정밀 검증 서비스는 후보마다 다음 값을 남겨야 합니다.

- 매칭 개수
- inlier 개수
- inlier 비율
- 재투영 오차
- PnP 성공 여부
- 추정 위치와 방향
- 최종 신뢰도

전역 검색 점수가 높더라도 inlier가 부족하거나 재투영 오차가 크면 “불확실” 또는 다중 후보로 반환합니다.
